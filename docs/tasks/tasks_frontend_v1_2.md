# 前端开发任务列表（按时间顺序）

本文档基于产品需求文档（PRD）及后端技术设计方案，拆解前端开发任务，按阶段组织。任务粒度以1小时为单位，便于进度跟踪。可并行执行的任务放在同一阶段内并列列出。

---

## 阶段一：项目初始化与基础架构（预计总工时：6h）

### 任务 1.1：项目脚手架搭建（2h）
- 使用 Vite + React + TypeScript 初始化项目。
- 配置 Tailwind CSS、PostCSS、Autoprefixer。
- 配置 ESLint、Prettier。
- 设置路径别名（如 `@/` 指向 `src/`）。
- 配置环境变量（API_BASE_URL、WS_URL 等）。

### 任务 1.2：基础 UI 组件库选型与配置（2h）
- 安装并配置 Vercel AI SDK（`ai`、`@ai-sdk/react`）。
- 安装并配置 `react-hook-form`（用于表单，如上传元数据）。
- 安装 `react-hot-toast` 或类似通知库。
- 安装 `react-dropzone`（用于拖拽上传区域）。

### 任务 1.3：全局状态管理与路由（2h）
- 使用 Zustand 或 React Context 管理用户状态（登录信息、AI昵称）。
- 安装 React Router，配置基础路由（首页、工作空间、设置等）。
- 创建布局组件（Header、Sidebar 可选）。

---

## 阶段二：核心对话界面（预计总工时：12h）

### 并行任务组 A（可并行执行，总工时：6h）

#### 任务 2.1：对话容器组件（2h）
- 实现 `ChatContainer` 组件，包含消息列表区域和输入区域。
- 使用 Vercel AI SDK 的 `useChat` hook 管理消息流。
- 实现消息列表滚动、自动滚动到底部。

#### 任务 2.2：消息渲染组件（2h）
- 实现 `Message` 组件，区分用户消息和 AI 消息样式。
- 支持 Markdown 渲染（`react-markdown`）。
- 支持代码高亮（`remark-gfm`、`rehype-highlight`）。

#### 任务 2.3：富文本标记与悬浮窗（2h）
- 实现 `<tool>` 和 `<rag>` 标签的解析与渲染。
- 使用 `react-popper` 或原生 CSS 实现悬浮窗，展示工具调用详情（搜索关键词、结果数量）或 RAG 检索片段。

### 并行任务组 B（可并行执行，总工时：6h）

#### 任务 2.4：输入框与发送功能（2h）
- 实现 `ChatInput` 组件，支持多行文本、回车发送、Shift+Enter 换行。
- 发送时禁用按钮，等待响应后恢复。
- 显示“正在输入”动画（AI 响应中）。

#### 任务 2.5：AI 昵称设置与用户信息收集（2h）
- 实现首次访问时的信息收集弹窗（姓名、邮箱）。
- 实现设置页面或对话中的 AI 昵称修改功能。
- 在对话界面头部显示 AI 昵称和用户姓名。

#### 任务 2.6：任务列表侧边栏（可选，2h）
- 实现侧边栏显示任务列表（简洁模式，可展开）。
- 点击任务可快速在对话中提及。
- 支持标记完成、删除等简单操作（通过调用后端 API）。

---

## 阶段三：个人工作空间（文件管理）（预计总工时：16h）

### 并行任务组 C（可并行执行，总工时：8h）

#### 任务 3.1：文件浏览器基础组件（3h）
- 实现 `FileBrowser` 组件，支持网格/列表视图切换。
- 实现文件卡片/行组件，展示图标、文件名、大小、上传时间、语义类型标签。
- 实现文件夹结构（可选，1.0版本可扁平化，但预留 UI）。

#### 任务 3.2：拖拽上传区域与进度反馈（3h）
- 集成 `react-dropzone`，实现工作空间任意区域的拖拽上传。
- 上传时显示虚线边框的占位节点，动态更新进度条。
- 上传成功后节点变为实线，刷新文件列表。
- 上传失败时节点变红，显示错误信息，提供重试按钮。

#### 任务 3.3：文件操作 UI（2h）
- 实现文件右键菜单或操作按钮（重命名、删除、下载）。
- 实现删除确认弹窗。
- 实现语义类型修改下拉选择（简历、学习资料、小抄等）。

### 并行任务组 D（可并行执行，总工时：8h）

#### 任务 3.4：文件上传元数据弹窗（2h）
- 实现弹窗组件，要求用户填写语义类型（必填，下拉或单选）。
- 支持用户输入自定义语义类型（可选）。
- 弹窗确认后开始上传。

#### 任务 3.5：文件列表与后端 API 对接（3h）
- 实现 `useFiles` hook 获取文件列表、上传、删除、重命名等操作。
- 实现分页或无限滚动（可选）。
- 处理加载状态、错误状态。

#### 任务 3.6：文件搜索与筛选（3h）
- 实现按文件名搜索输入框。
- 实现按语义类型筛选的下拉框。
- 实现排序（按时间、名称、大小）。

---

## 阶段四：系统集成与用户体验优化（预计总工时：10h）

### 并行任务组 E（可并行执行，总工时：10h）

#### 任务 4.1：全局通知与错误处理（2h）
- 集成 `react-hot-toast`，统一处理 API 错误、上传失败等。
- 实现系统通知（浏览器通知）用于文件上传成功/失败。

#### 任务 4.2：用户认证与会话管理（2h）
- 实现简单的 token 或 session 管理（可与后端协商使用 `Authorization` 头）。
- 实现登录页面（或与信息收集整合）。
- 处理未登录状态跳转。

#### 任务 4.3：响应式与移动端适配（2h）
- 使用 Tailwind 的响应式工具，确保在移动端（最小宽度 375px）基本可用。
- 调整对话界面布局（侧边栏可折叠）。

#### 任务 4.4：性能优化与代码分割（2h）
- 使用 React.lazy 对工作空间、对话界面等模块进行代码分割。
- 优化图片加载（懒加载）。
- 使用 `useMemo`、`useCallback` 避免不必要的重渲染。

#### 任务 4.5：本地存储与离线缓存（2h）
- 将用户信息、AI昵称缓存到 localStorage，减少初始请求。
- 将文件列表缓存，支持离线查看（可选）。

---

## 阶段五：测试与部署（预计总工时：8h）

### 并行任务组 F（可并行执行，总工时：8h）

#### 任务 5.1：单元测试与组件测试（3h）
- 使用 Vitest + React Testing Library 测试核心组件。
- 测试对话消息渲染、富文本标记解析。
- 测试文件上传流程（mock 后端 API）。

#### 任务 5.2：端到端测试（3h）
- 使用 Playwright 模拟用户完整流程：
  - 首次访问输入姓名邮箱。
  - 对话、创建任务。
  - 上传文件并验证 RAG 检索。
  - 修改 AI 昵称。

#### 任务 5.3：构建与部署（2h）
- 配置 Vite 生产构建，优化输出。
- 部署到 Cloudflare Pages 或 Vercel。
- 配置自定义域名（可选）。
- 配置环境变量（API 地址）。

## 并行执行建议

- **阶段二** 中，任务组 A（对话容器、消息渲染、富文本标记）可由 2 人并行开发，组 B（输入框、用户信息、任务侧边栏）由另 2 人并行。
- **阶段三** 中，任务组 C（文件浏览器基础、拖拽上传、文件操作）与组 D（元数据弹窗、API 对接、搜索筛选）可并行。
- **阶段四** 和 **阶段五** 可在功能开发的同时并行进行。

此任务列表可根据实际团队配置灵活调整，确保每个任务粒度可追踪。
